## 🧠 AI/ML 前沿论文

**[Solvita: Enhancing Large Language Models for Competitive Programming via Agentic Evolution](https://huggingface.co/papers/2605.15301)**
🔬 突破：推翻“LLM在编程竞赛中只能靠静态知识”的假设——Solvita通过**将每次解题的经验（策略选择、调试过程、失败模式）存入外部记忆，并在后续任务中检索复用**，使LLM在Codeforces上的通过率从基线（单Agent）的18%提升至34%，接近人类业余选手水平。关键量化：在2400+难度题目上，Solvita比静态多Agent框架（如ChatDev）的通过率高12个百分点。
⚙️ 工程影响：这意味着**Agent框架需要从“无状态”转向“有状态”**——每次推理的中间结果（如错误堆栈、调试日志）不再是垃圾，而是可复用的“经验”。对部署的影响：需要引入持久化记忆存储（如向量数据库）和检索机制，增加约20%的推理延迟，但能显著减少重复错误。

**[PAGER: Bridging the Semantic-Execution Gap in Point-Precise Geometric GUI Control](https://huggingface.co/papers/2605.15963)**
🔬 突破：推翻“GUI Agent只需要区域级点击”的假设——PAGER发现，在Figma、CAD等精确几何工具中，现有Agent（如GPT-4V）的点击误差中位数达15像素，导致拓扑错误（如矩形变梯形）的概率为37%。PAGER通过**引入“几何依赖图”和“点精确校正网络”**，将点击误差降至2像素以内，拓扑错误率降至4%。
⚙️ 工程影响：对GUI Agent的部署意味着**不能再用“截图+坐标”的粗粒度方案**——需要引入像素级对齐和几何约束推理。具体影响：推理管线需增加一个“几何校正”步骤（约50ms/次），但能解锁对Figma、AutoCAD等精确工具的自动化操作能力。

**[Agentic Discovery of Neural Architectures: AIRA-Compose and AIRA-Design](https://huggingface.co/papers/2605.15871)**
🔬 突破：推翻“神经网络架构搜索（NAS）需要大量GPU算力”的假设——AIRA-Compose用11个LLM Agent在24小时内探索了14种新架构（包括Transformer-Mamba混合体），其中一种在3B参数规模下，推理速度比同等规模Transformer快1.7倍，而搜索成本仅为传统NAS（如ENAS）的1/50。
⚙️ 工程影响：这意味着**架构搜索不再是大公司的专利**——任何团队都可以用LLM Agent在24小时内设计出领域特化架构。对部署的影响：Agent生成的架构可能不兼容现有推理框架（如vLLM），需要额外的算子适配成本（约2-3人周）。

## ⚡ 技术范式变化信号

**[从“无状态Agent”到“有状态Agent”的范式转移]**：Solvita论文和Decepticon（昨日）共同指向一个趋势——Agent不再是一次性推理，而是通过外部记忆（经验库、攻击路径记录）实现跨任务学习。为什么是现在：LLM的上下文窗口已从4K扩展到1M，但“一次性推理”的瓶颈不在上下文长度，而在无法复用历史经验。对工程决策的直接影响：所有Agent框架（LangChain、AutoGPT）都需要引入持久化记忆层，否则将在复杂任务（如编程竞赛、渗透测试）中被有状态Agent碾压。

**[本地AI从“单点工具”走向“集成平台”]**：DreamServer、LEANN、whichllm三个项目在同一天出现，标志着本地AI正在从“一个工具做一件事”（Ollama做推理、ChromaDB做RAG）转向“一个平台做所有事”。为什么是现在：消费级GPU（RTX 4090）的显存已达24GB，足以同时运行推理+RAG+图像生成，但缺乏统一的资源调度层。对工程决策的直接影响：团队在构建本地AI应用时，应优先评估集成平台（如DreamServer）而非拼凑单点工具，以降低跨模块通信的延迟和复杂度。

**[编码Agent的“技能可移植性”成为新战场]**：skillkit和Semble（HN）都试图解决同一个问题——如何让Agent的技能/知识在不同工具间复用。为什么是现在：编码Agent从“单一工具”（Copilot）扩展到“多工具生态”（Claude Code、Cursor、Codex），但每个工具的技能是孤立的。对工程决策的直接影响：团队应开始将内部最佳实践封装为“技能包”（如skillkit格式），而非绑定在单一Agent工具上，以避免未来迁移成本。

## 🧠 AI/ML Frontier Papers

**[Solvita: Enhancing Large Language Models for Competitive Programming via Agentic Evolution](https://huggingface.co/papers/2605.15301)**
🔬 Breakthrough: Overturns the assumption that "LLMs can only rely on static knowledge in programming competitions"—Solvita **stores the experience from each problem-solving session (strategy selection, debugging process, failure modes) in external memory and retrieves it for reuse in subsequent tasks**, boosting the LLM's pass rate on Codeforces from 18% (single Agent baseline) to 34%, approaching the level of amateur human competitors. Key metric: On problems with difficulty 2400+, Solvita outperforms static multi-agent frameworks (e.g., ChatDev) by 12 percentage points.
⚙️ Engineering Impact: This means **Agent frameworks need to shift from "stateless" to "stateful"**—intermediate results from each inference (e.g., error stacks, debug logs) are no longer garbage but reusable "experience." Deployment implications: Requires introducing persistent memory storage (e.g., vector databases) and retrieval mechanisms, adding about 20% inference latency, but significantly reducing repeated errors.

**[PAGER: Bridging the Semantic-Execution Gap in Point-Precise Geometric GUI Control](https://huggingface.co/papers/2605.15963)**
🔬 Breakthrough: Overturns the assumption that "GUI Agents only need region-level clicks"—PAGER finds that in precise geometric tools like Figma and CAD, existing Agents (e.g., GPT-4V) have a median click error of 15 pixels, leading to a 37% probability of topological errors (e.g., rectangles becoming trapezoids). PAGER **introduces a "geometric dependency graph" and a "point-precise correction network"** to reduce click error to under 2 pixels and topological error rate to 4%.
⚙️ Engineering Impact: For GUI Agent deployment, this means **the coarse-grained "screenshot + coordinates" approach is no longer viable**—pixel-level alignment and geometric constraint reasoning are needed. Specific impact: The inference pipeline needs an additional "geometric correction" step (about 50ms per operation), but this unlocks the ability to automate precise tools like Figma and AutoCAD.

**[Agentic Discovery of Neural Architectures: AIRA-Compose and AIRA-Design](https://huggingface.co/papers/2605.15871)**
🔬 Breakthrough: Overturns the assumption that "Neural Architecture Search (NAS) requires massive GPU compute"—AIRA-Compose uses 11 LLM Agents to explore 14 new architectures (including Transformer-Mamba hybrids) within 24 hours. One of these, at the 3B parameter scale, achieves 1.7x faster inference than a similarly sized Transformer, while the search cost is only 1/50th of traditional NAS (e.g., ENAS).
⚙️ Engineering Impact: This means **architecture search is no longer exclusive to large companies**—any team can design domain-specific architectures within 24 hours using LLM Agents. Deployment implications: Agent-generated architectures may not be compatible with existing inference frameworks (e.g., vLLM), requiring additional operator adaptation costs (about 2-3 person-weeks).

## ⚡ Signals of Technological Paradigm Shift

**[Paradigm Shift from "Stateless Agent" to "Stateful Agent"]**: The Solvita paper and Decepticon (yesterday) both point to a trend—Agents are no longer one-shot inference but achieve cross-task learning through external memory (experience bases, attack path records). Why now: LLM context windows have expanded from 4K to 1M, but the bottleneck of "one-shot inference" is not context length, but the inability to reuse historical experience. Direct Impact on Engineering Decisions: All Agent frameworks (LangChain, AutoGPT) need to introduce a persistent memory layer, or they will be crushed by stateful Agents in complex tasks (e.g., programming competitions, penetration testing).

**[Local AI Moving from "Single-Point Tools" to "Integrated Platforms"]**: The emergence of DreamServer, LEANN, and whichllm on the same day signals that local AI is shifting from "one tool for one job" (Ollama for inference, ChromaDB for RAG) to "one platform for everything." Why now: Consumer-grade GPUs (RTX 4090) now have 24GB of VRAM, enough to run inference + RAG + image generation simultaneously, but lack a unified resource scheduling layer. Direct Impact on Engineering Decisions: When building local AI applications, teams should prioritize evaluating integrated platforms (e.g., DreamServer) over patching together single-point tools to reduce the latency and complexity of cross-module communication.

**["Skill Portability" of Coding Agents Becomes a New Battleground]**: Both skillkit and Semble (HN) attempt to solve the same problem—how to make an Agent's skills/knowledge reusable across different tools. Why now: Coding Agents have expanded from "single tools" (Copilot) to a "multi-tool ecosystem" (Claude Code, Cursor, Codex), but each tool's skills are siloed. Direct Impact on Engineering Decisions: Teams should start packaging internal best practices as "skill packages" (e.g., skillkit format) rather than binding them to a single Agent tool, to avoid future migration costs.

## ⚡ 技术范式变化信号

**[「ステートレスエージェント」から「ステートフルエージェント」へのパラダイムシフト]**：Solvita論文とDecepticon（昨日）は共通のトレンドを示しています。エージェントはもはや一度きりの推論ではなく、外部メモリ（経験データベース、攻撃パス記録）を通じてタスク間学習を実現します。なぜ今なのか：LLMのコンテキストウィンドウは4Kから1Mに拡張されましたが、「一度きりの推論」のボトルネックはコンテキスト長ではなく、過去の経験を再利用できないことにあります。エンジニアリング上の意思決定への直接的な影響：すべてのエージェントフレームワーク（LangChain、AutoGPT）は永続的なメモリレイヤーを導入する必要があります。そうしなければ、複雑なタスク（プログラミングコンテスト、ペネトレーションテストなど）において、ステートフルエージェントに圧倒されるでしょう。

**[ローカルAIが「単一目的ツール」から「統合プラットフォーム」へ]**：DreamServer、LEANN、whichllmの3つのプロジェクトが同じ日に登場したことは、ローカルAIが「1つのツールが1つのことを行う」（Ollamaは推論、ChromaDBはRAG）から「1つのプラットフォームがすべてを行う」へと移行していることを示しています。なぜ今なのか：コンシューマー向けGPU（RTX 4090）のVRAMは24GBに達し、推論+RAG+画像生成を同時に実行するのに十分ですが、統一されたリソーススケジューリングレイヤーが不足しています。エンジニアリング上の意思決定への直接的な影響：チームはローカルAIアプリケーションを構築する際、モジュール間の通信レイテンシと複雑さを低減するために、単一目的ツールを寄せ集めるのではなく、統合プラットフォーム（DreamServerなど）を優先的に評価する必要があります。

**[コーディングエージェントの「スキルの移植性」が新たな戦場に]**：skillkitとSemble（HN）はどちらも同じ問題を解決しようとしています。つまり、エージェントのスキル/知識を異なるツール間でどのように再利用するかということです。なぜ今なのか：コーディングエージェントは「単一ツール」（Copilot）から「マルチツールエコシステム」（Claude Code、Cursor、Codex）へと拡大しましたが、各ツールのスキルは孤立しています。エンジニアリング上の意思決定への直接的な影響：チームは、将来の移行コストを回避するために、内部のベストプラクティスを単一のエージェントツールにバインディングするのではなく、「スキルパッケージ」（skillkit形式など）としてカプセル化し始めるべきです。
