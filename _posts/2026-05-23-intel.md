## 🧠 AI/ML 前沿论文

**[LoREnc: Low-Rank Encryption for Securing Foundation Models and LoRA Adapters](https://huggingface.co/papers/2605.13163)**
🔬 突破：推翻了“保护模型权重必须重训练或依赖原始数据集”的假设。LoREnc 通过“谱截断+补偿”机制，在无需访问训练数据的情况下，将模型权重的低秩成分加密，同时为授权适配器提供补偿信息。实验表明，该方法能将模型恢复攻击的成功率从 95% 降至 5% 以下，且对模型精度的影响小于 0.5%。
⚙️ 工程影响：这意味着部署 LoRA 适配器时，不再需要担心适配器泄露导致基座模型被逆向。团队可以在公开仓库中分发加密后的基座模型，仅向授权用户提供解密密钥和补偿适配器，无需改变现有推理流程。

**[AutoRubric-T2I: Robust Rule-Based Reward Model for Text-to-Image Alignment](https://huggingface.co/papers/2605.17602)**
🔬 突破：推翻了“图像奖励模型必须基于 Bradley-Terry 偏好模型训练”的假设。AutoRubric-T2I 使用 VLM 自动生成评分规则（rubric），然后基于规则进行确定性评分，而非学习人类偏好。在 T2I 对齐任务中，其评分与人类判断的相关性（Spearman ρ=0.82）超过了基于偏好训练的模型（如 ImageReward，ρ=0.76），且训练成本降低约 90%。
⚙️ 工程影响：这意味着团队可以快速为特定领域（如医学图像、游戏原画）定制奖励模型，只需提供领域描述和少量示例，无需收集大量人工偏好数据。推理时，该模型可作为 Stable Diffusion 的“过滤器”，拒绝低分生成结果。

**[Efficient Agentic Reasoning Through Self-Regulated Simulative Planning](https://huggingface.co/papers/2605.22138)**
🔬 突破：推翻了“Agent 应该始终使用 Chain-of-Thought 推理”的假设。论文提出将 Agent 决策分解为三个系统：反应式（System I）、模拟推理（System II）和自调节（System III）。其中 System III 负责决定何时调用 System II（即何时进行深度规划）。实验表明，该方法在 WebArena 基准上，推理 token 消耗减少约 60%，同时任务成功率提升约 5%。
⚙️ 工程影响：这意味着 Agent 框架不应默认启用“思考-行动”循环，而应引入一个“元控制器”来决定何时需要规划。团队可以在 LangGraph 或 CrewAI 中实现一个简单的“规划触发器”（如任务复杂度超过阈值时），以节省推理成本。

## 🧠 AI/ML Frontier Papers

**[LoREnc: Low-Rank Encryption for Securing Foundation Models and LoRA Adapters](https://huggingface.co/papers/2605.13163)**
🔬 Breakthrough: Overturns the assumption that "protecting model weights requires retraining or access to the original dataset." LoREnc uses a "spectral truncation + compensation" mechanism to encrypt the low-rank components of model weights without needing access to training data, while providing compensation information for authorized adapters. Experiments show this method reduces the success rate of model recovery attacks from 95% to below 5%, with less than 0.5% impact on model accuracy.
⚙️ Engineering Impact: This means when deploying LoRA adapters, there is no longer a need to worry about adapter leakage leading to base model inversion. Teams can distribute encrypted base models in public repositories, providing decryption keys and compensation adapters only to authorized users, without altering existing inference pipelines.

**[AutoRubric-T2I: Robust Rule-Based Reward Model for Text-to-Image Alignment](https://huggingface.co/papers/2605.17602)**
🔬 Breakthrough: Overturns the assumption that "image reward models must be trained based on the Bradley-Terry preference model." AutoRubric-T2I uses a VLM to automatically generate scoring rules (rubrics) and then performs deterministic scoring based on those rules, rather than learning human preferences. In T2I alignment tasks, its scoring correlation with human judgment (Spearman ρ=0.82) surpasses preference-trained models (e.g., ImageReward, ρ=0.76), while training costs are reduced by approximately 90%.
⚙️ Engineering Impact: This means teams can quickly customize reward models for specific domains (e.g., medical images, game concept art) by providing a domain description and a few examples, without needing to collect large amounts of human preference data. During inference, this model can serve as a "filter" for Stable Diffusion, rejecting low-scoring generated results.

**[Efficient Agentic Reasoning Through Self-Regulated Simulative Planning](https://huggingface.co/papers/2605.22138)**
🔬 Breakthrough: Overturns the assumption that "Agents should always use Chain-of-Thought reasoning." The paper proposes decomposing Agent decision-making into three systems: Reactive (System I), Simulative Reasoning (System II), and Self-Regulation (System III). System III is responsible for deciding when to invoke System II (i.e., when deep planning is needed). Experiments show this method reduces reasoning token consumption by approximately 60% on the WebArena benchmark while improving task success rates by about 5%.
⚙️ Engineering Impact: This means Agent frameworks should not default to a "think-act" loop but should introduce a "meta-controller" to decide when planning is necessary. Teams can implement a simple "planning trigger" (e.g., when task complexity exceeds a threshold) in LangGraph or CrewAI to save reasoning costs.

⚙️ 工程影響：這意味著團隊可以快速為特定領域（如醫學圖像、遊戲原畫）客製獎勵模型，只需提供領域描述和少量範例，無需收集大量人工偏好資料。推理時，該模型可作為 Stable Diffusion 的「過濾器」，拒絕低分生成結果。

**[Efficient Agentic Reasoning Through Self-Regulated Simulative Planning](https://huggingface.co/papers/2605.22138)**
🔬 突破：推翻了「Agent 應該始終使用 Chain-of-Thought 推理」的假設。論文提出將 Agent 決策分解為三個系統：反應式（System I）、模擬推理（System II）和自調節（System III）。其中 System III 負責決定何時調用 System II（即何時進行深度規劃）。實驗表明，該方法在 WebArena 基準上，推理 token 消耗減少約 60%，同時任務成功率提升約 5%。
⚙️ 工程影響：這意味著 Agent 框架不應預設啟用「思考-行動」循環，而應引入一個「元控制器」來決定何時需要規劃。團隊可以在 LangGraph 或 CrewAI 中實現一個簡單的「規劃觸發器」（如任務複雜度超過閾值時），以節省推理成本。
