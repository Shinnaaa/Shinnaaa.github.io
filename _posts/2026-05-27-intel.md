## 🧠 AI/ML 前沿论文

**[MobileMoE: Scaling On-Device Mixture of Experts](https://huggingface.co/papers/2605.27358)**
🔬 突破：推翻了“MoE架构在<1B参数规模无优势”的假设。通过提出“on-device MoE scaling law”，在0.3-0.9B激活参数下，MobileMoE在手机端（如iPhone 15）的推理延迟（<50ms/token）和内存占用（<2GB）均优于同参数量Dense模型（如Gemma 2B），且MMLU得分高出约8%。
⚙️ 工程影响：意味着移动端LLM部署不再只能选择“小Dense模型”或“大模型云端调用”。MobileMoE的“稀疏激活”特性允许开发者将模型总参数量做到5.3B，但每次推理只激活0.9B，从而在手机本地运行接近GPT-3.5级别的模型，且无需联网。

**[MRT: Masked Region Transformer for Layered Image Generation and Editing at Scale](https://huggingface.co/papers/2605.27235)**
🔬 突破：将图像生成从“单层像素”推进到“多层透明图层”的端到端生成。MRT（20B参数）直接输出RGBA多层图像，而非像Stable Diffusion那样先生成单层再手动抠图。在10M多语言设计样本上训练，支持对任意图层进行“替换、删除、重排”等编辑操作，且保持其他图层不变。
⚙️ 工程影响：对UI/UX设计、广告物料生成等场景是直接生产力工具。现有工作流（生成→Photoshop抠图→分层编辑）可被MRT一步替代。但20B参数意味着推理需要至少40GB显存（如A100），且生成多层图像的速度约为单层模型的2-3倍。

**[Efficient Agentic Reinforcement Learning with On-Policy Intrinsic Knowledge Boundary Enhancement](https://huggingface.co/papers/2605.26952)**
🔬 突破：揭示了Agent RL训练中的一个关键问题——模型会“忘记”自己何时不需要调用工具，导致冗余工具调用。AKBE通过引入“内在知识边界”奖励，让模型学会区分“我知道答案”和“我需要查工具”，而非像现有方法（如奖励塑形）那样简单惩罚所有工具调用。实验显示，AKBE在ToolBench上使工具调用准确率提升12%，同时减少30%的冗余调用。
⚙️ 工程影响：直接提升Agent的效率和成本。对于部署了Agent的企业，AKBE意味着每次任务可减少约30%的API调用费用（如搜索、数据库查询），且Agent在简单问题上响应更快（因为直接回答而非调用工具）。

## 🧠 AI/ML Frontier Papers

**[MobileMoE: Scaling On-Device Mixture of Experts](https://huggingface.co/papers/2605.27358)**
🔬 Breakthrough: Overturns the assumption that "MoE architecture has no advantage at <1B parameter scale." By proposing an "on-device MoE scaling law," MobileMoE achieves inference latency (<50ms/token) and memory usage (<2GB) on mobile devices (e.g., iPhone 15) that are superior to Dense models with the same number of active parameters (e.g., Gemma 2B), while scoring about 8% higher on MMLU, using 0.3-0.9B active parameters.
⚙️ Engineering Impact: This means mobile LLM deployment is no longer limited to choosing between "small Dense models" or "large models via cloud calls." MobileMoE's "sparse activation" feature allows developers to have a total model size of 5.3B parameters, but only activate 0.9B per inference, enabling running a model close to GPT-3.5 level locally on a phone without needing an internet connection.

**[MRT: Masked Region Transformer for Layered Image Generation and Editing at Scale](https://huggingface.co/papers/2605.27235)**
🔬 Breakthrough: Advances image generation from "single-layer pixels" to end-to-end generation of "multi-layer transparent layers." MRT (20B parameters) directly outputs RGBA multi-layer images, unlike Stable Diffusion which generates a single layer first and then requires manual matting. Trained on 10M multi-language design samples, it supports editing operations like "replace, delete, reorder" on any layer while keeping other layers unchanged.
⚙️ Engineering Impact: This is a direct productivity tool for scenarios like UI/UX design and ad material generation. The existing workflow (generate → Photoshop matting → layered editing) can be replaced by MRT in one step. However, the 20B parameters mean inference requires at least 40GB of VRAM (e.g., A100), and generating multi-layer images is about 2-3 times slower than single-layer models.

**[Efficient Agentic Reinforcement Learning with On-Policy Intrinsic Knowledge Boundary Enhancement](https://huggingface.co/papers/2605.26952)**
🔬 Breakthrough: Reveals a key problem in Agent RL training—the model "forgets" when it doesn't need to call a tool, leading to redundant tool calls. AKBE introduces an "intrinsic knowledge boundary" reward, teaching the model to distinguish between "I know the answer" and "I need to check a tool," rather than simply penalizing all tool calls like existing methods (e.g., reward shaping). Experiments show AKBE improves tool call accuracy by 12% on ToolBench while reducing redundant calls by 30%.
⚙️ Engineering Impact: Directly improves agent efficiency and cost. For enterprises deploying agents, AKBE means a potential ~30% reduction in API call costs (e.g., search, database queries) per task, and agents respond faster on simple questions (by answering directly instead of calling a tool).
