## 🧠 AI/ML 前沿论文

**[SANA-Streaming: Real-time Streaming Video Editing with Hybrid Diffusion Transformer](https://huggingface.co/papers/2605.30409)**
🔬 突破：现有视频编辑模型（如Runway Gen-3、Pika）无法做到实时流式处理，延迟通常在秒级。SANA-Streaming通过“系统-算法协同设计”首次在消费级GPU（RTX 4090）上实现了**实时（>30fps）的高分辨率（1024x576）视频编辑**。核心创新是Hybrid Diffusion Transformer架构，在部分block中引入softmax attention以增强局部建模能力，而非像现有DiT那样全部使用全局attention。
⚙️ 工程影响：这意味着直播场景中的实时视频风格迁移、背景替换、物体移除成为可能。团队可评估其能否替代现有的“先录制后编辑”工作流，但需注意其依赖的GPU显存（约16GB）和模型大小（约3B参数）。

**[dMoE: dLLMs with Learnable Block Experts](https://huggingface.co/papers/2605.30876)**
🔬 突破：现有Diffusion LLM（dLLM）与MoE架构结合时存在根本性矛盾——dLLM的并行解码处理多个token，而MoE的token级路由破坏了这种并行性。dMoE提出“可学习块专家”（Learnable Block Experts），将专家路由从token级提升到block级，使得**dLLM的并行解码效率提升约2倍**，同时模型容量（参数量）增加约3倍。
⚙️ 工程影响：对于正在探索非自回归LLM的团队，dMoE提供了一条将MoE扩展到dLLM的可行路径。但需注意，其训练复杂度较高，且block级专家可能不如token级专家灵活。

**[MAAT: Multi-phase Adapter-Aware Targeted Unlearning](https://huggingface.co/papers/2605.30514)**
🔬 突破：现有机器遗忘（Machine Unlearning）评估基准存在严重偏差——因果类问题（Why）占比不足1.3%，导致许多方法在因果知识上失败却无法被检测。论文提出5WBENCH基准（5000样本，每类1000），首次量化了现有遗忘方法在“Why”类问题上的失败率：**主流方法（如SSD）在Why类问题上的遗忘成功率仅为12%，而在What类上为85%**。
⚙️ 工程影响：这意味着所有声称“有效”的遗忘方法都需要在5WBENCH上重新评估。团队在构建或选择遗忘方案时，必须将因果知识的遗忘效果作为独立指标，而非只看聚合分数。

## 🧠 AI/ML Frontier Papers

**[SANA-Streaming: Real-time Streaming Video Editing with Hybrid Diffusion Transformer](https://huggingface.co/papers/2605.30409)**
🔬 Breakthrough: Existing video editing models (like Runway Gen-3, Pika) cannot achieve real-time streaming processing, with latency typically in seconds. SANA-Streaming achieves **real-time (>30fps) high-resolution (1024x576) video editing** on a consumer-grade GPU (RTX 4090) for the first time through "system-algorithm co-design". The core innovation is the Hybrid Diffusion Transformer architecture, which introduces softmax attention in some blocks to enhance local modeling capabilities, rather than using global attention in all blocks like existing DiT.
⚙️ Engineering Impact: This makes real-time video style transfer, background replacement, and object removal in live streaming scenarios possible. Teams can evaluate whether it can replace the existing "record first, edit later" workflow, but need to be aware of its GPU memory requirements (approximately 16GB) and model size (approximately 3B parameters).

**[dMoE: dLLMs with Learnable Block Experts](https://huggingface.co/papers/2605.30876)**
🔬 Breakthrough: There is a fundamental contradiction when combining existing Diffusion LLM (dLLM) with the MoE architecture — dLLM's parallel decoding processes multiple tokens, while MoE's token-level routing destroys this parallelism. dMoE proposes "Learnable Block Experts", elevating expert routing from the token level to the block level, thereby **improving the parallel decoding efficiency of dLLM by approximately 2x**, while increasing model capacity (parameter count) by approximately 3x.
⚙️ Engineering Impact: For teams exploring non-autoregressive LLMs, dMoE provides a feasible path to extend MoE to dLLMs. However, it should be noted that its training complexity is higher, and block-level experts may be less flexible than token-level experts.

**[MAAT: Multi-phase Adapter-Aware Targeted Unlearning](https://huggingface.co/papers/2605.30514)**
🔬 Breakthrough: Existing evaluation benchmarks for Machine Unlearning have a severe bias — causal questions (Why) account for less than 1.3%, causing many methods to fail on causal knowledge without being detected. The paper proposes the 5WBENCH benchmark (5000 samples, 1000 per category), which for the first time quantifies the failure rate of existing unlearning methods on "Why" type questions: **the unlearning success rate of mainstream methods (like SSD) on Why-type questions is only 12%, compared to 85% on What-type questions**.
⚙️ Engineering Impact: This means all methods claiming to be "effective" for unlearning need to be re-evaluated on 5WBENCH. When building or selecting an unlearning solution, teams must treat the unlearning effectiveness of causal knowledge as an independent metric, rather than just looking at aggregate scores.

## ⚡ Signals of Technological Paradigm Shift

**[Agent UI shifting from "chat interface" to "tool call visualization"]**: The simultaneous emergence of Hermes WebUI and Sandcastle signals that the Agent interaction paradigm is shifting from "black-box chat" to "white-box toolchain". Engineers need to rethink: An Agent's UI should not just be a chat box, but a "tool call console + thought chain debugger". This means for all teams building Agent products, UI design priorities should shift from "conversational fluency" to "operational transparency and debuggability".

**[Video editing shifting from "offline batch processing" to "real-time streaming"]**: SANA-Streaming achieving real-time video editing on a consumer-grade GPU, combined with the emergence of Dav2d (AV2 decoder), indicates that the video processing pipeline is shifting from "record first, edit later" to "edit while live streaming". For the engineering architecture of live streaming, gaming, remote collaboration, and other scenarios, this means: video codec, AI inference, and rendering pipelines need to be integrated into the same real-time pipeline, rather than being processed in stages as is currently done.

**[1-bit quantization shifting from "academic toy" to "production-ready"]**: Bonsai's 4B parameter 1-bit image generation model reaching a usable level on a phone indicates that 1-bit quantization is no longer an experimental technology with excessive accuracy loss. For edge AI deployment, this means: the "sweet spot" for model compression may be shifting from 4-bit/8-bit to 1-bit, but its performance in NLP (e.g., LLMs) needs to be verified as equally effective. In terms of engineering decisions, one should start evaluating the maturity of 1-bit quantization toolchains (like BitNet, Bonsai).
