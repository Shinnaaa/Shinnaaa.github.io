## 🧠 AI/ML 前沿论文

**[WebRISE: Requirement-Induced State Evaluation for MLLM-Generated Web Artifacts](https://huggingface.co/papers/2606.03220)**
🔬 突破：现有MLLM生成网页的评测（如WebArena、VisualWebBench）只检查页面元素是否存在，但WebRISE通过**Interaction Contract Graphs（ICGs）**将用户需求转化为可执行的“状态-转换”断言，覆盖442个任务、5,495次状态转换。相比WebArena的~200个任务，WebRISE的评测粒度提升了27倍，且能检测到“按钮存在但点击后无反应”这类功能性问题。
⚙️ 工程影响：对前端Agent开发团队意味着：评测标准从“页面看起来对不对”升级为“页面用起来对不对”。建议将WebRISE的ICG模式引入自己的Agent评测管道，替代现有的截图对比方案。

**[ThoughtFold: Folding Reasoning Chains via Introspective Preference Learning](https://huggingface.co/papers/2606.03503)**
🔬 突破：现有RLVR方法（如DeepSeek-R1）会强化长CoT中的冗余探索，导致“过度思考”。ThoughtFold通过**内省偏好学习**让模型学会区分“有效推理步骤”和“试错步骤”，在GSM8K上以50%的推理步数达到同等准确率。对比DeepSeek-R1的“结果奖励+长度惩罚”，ThoughtFold的“过程级偏好信号”使推理效率提升2倍。
⚙️ 工程影响：对部署推理模型的团队意味着：可以在不降低准确率的前提下，将推理延迟减半。建议在现有的推理服务中尝试用ThoughtFold的偏好数据微调，替代简单的“长度惩罚”策略。

**[AutoLab: Can Frontier Models Solve Long-Horizon Auto Research and Engineering Tasks?](https://huggingface.co/papers/2606.05080)**
🔬 突破：现有Agent基准（如SWE-bench、GAIA）只评测单步或短周期任务，AutoLab包含36个“长周期闭环优化”任务（如“优化一个编译器的代码生成阶段”），每个任务需要模型进行“提议→实验→测量→迭代”的循环。在测试中，最强的Claude 4仅完成12/36个任务，成功率33%，暴露了当前模型在“持续改进”上的根本性短板。
⚙️ 工程影响：对构建Agent平台的团队意味着：现有Agent框架（如LangGraph、AutoGen）的“线性执行”模式不适合长周期任务。需要引入“实验管理”和“结果回溯”组件，而非简单的“工具调用链”。

## 🧠 AI/ML Frontier Papers

**[WebRISE: Requirement-Induced State Evaluation for MLLM-Generated Web Artifacts](https://huggingface.co/papers/2606.03220)**  
🔬 Breakthrough: Existing MLLM-generated web page evaluations (e.g., WebArena, VisualWebBench) only check whether page elements exist, but WebRISE uses **Interaction Contract Graphs (ICGs)** to convert user requirements into executable "state-transition" assertions, covering 442 tasks and 5,495 state transitions. Compared to WebArena's ~200 tasks, WebRISE's evaluation granularity is 27 times finer, and it can detect functional issues like "button exists but no response on click."  
⚙️ Engineering Impact: For frontend Agent development teams, this means evaluation standards upgrade from "does the page look right" to "does the page work right." It is recommended to introduce WebRISE's ICG pattern into your own Agent evaluation pipeline, replacing existing screenshot comparison methods.

**[ThoughtFold: Folding Reasoning Chains via Introspective Preference Learning](https://huggingface.co/papers/2606.03503)**  
🔬 Breakthrough: Existing RLVR methods (e.g., DeepSeek-R1) reinforce redundant exploration in long CoTs, leading to "overthinking." ThoughtFold uses **introspective preference learning** to help the model distinguish between "effective reasoning steps" and "trial-and-error steps," achieving equal accuracy on GSM8K with 50% of the reasoning steps. Compared to DeepSeek-R1's "result reward + length penalty," ThoughtFold's "process-level preference signal" improves reasoning efficiency by 2x.  
⚙️ Engineering Impact: For teams deploying reasoning models, this means inference latency can be halved without sacrificing accuracy. It is recommended to fine-tune existing models with ThoughtFold's preference data, replacing simple "length penalty" strategies.

**[AutoLab: Can Frontier Models Solve Long-Horizon Auto Research and Engineering Tasks?](https://huggingface.co/papers/2606.05080)**  
🔬 Breakthrough: Existing Agent benchmarks (e.g., SWE-bench, GAIA) only evaluate single-step or short-cycle tasks. AutoLab includes 36 "long-horizon closed-loop optimization" tasks (e.g., "optimize the code generation phase of a compiler"), each requiring the model to perform a "propose → experiment → measure → iterate" cycle. In tests, the strongest Claude 4 completed only 12/36 tasks, with a 33% success rate, exposing a fundamental weakness in current models for "continuous improvement."  
⚙️ Engineering Impact: For teams building Agent platforms, this means existing Agent frameworks (e.g., LangGraph, AutoGen) with "linear execution" patterns are unsuitable for long-horizon tasks. "Experiment management" and "result backtracking" components need to be introduced, rather than simple "tool call chains."
