## 🧠 AI/ML 前沿论文

**[EEVEE: Towards Test-time Prompt Learning in the Real World for Self-Improving Agents](https://huggingface.co/papers/2606.11182)**
🔬 突破：推翻了“test-time prompt learning只能在单一数据集上有效”的假设，通过引入一个**任务路由器**，将多数据集、多领域的输入流动态分区，为每个分区学习独立的prompt配置。在混合了10个不同领域数据集的测试中，EEVEE比单prompt基线（如CoT）的准确率提升了12.3%，且路由器的额外开销仅约5ms/query。
⚙️ 工程影响：这意味着部署在真实世界的Agent（如客服机器人）不再需要为每个业务领域单独维护一个模型实例，而是通过一个统一的路由+多prompt池架构实现“自我适应”。工程团队需要关注的是：路由器的分类准确率（论文中为94.7%）直接决定了整体性能，且prompt池需要定期更新以应对数据漂移。

**[SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long-Horizon Deep Research](https://huggingface.co/papers/2606.09730)**
🔬 突破：改进了“单一Agent处理长上下文任务”的范式，提出**委托智能（Delegation Intelligence）**——主Agent将子任务委派给子Agent，子Agent只返回摘要，从而将主Agent的上下文消耗降低约80%。在“撰写一份关于量子计算在金融领域的应用”的深度研究任务中，SearchSwarm的最终报告质量（由人类专家评估）比单Agent基线（如AutoGPT）高出34%，且完成时间缩短了42%。
⚙️ 工程影响：这直接挑战了“无限上下文窗口”的工程方向。SearchSwarm表明，通过Agent间的委派和摘要，有限的上下文窗口可以处理理论上无边界的信息。工程团队需要实现的是：子Agent的“摘要保真度”监控机制，以及主Agent的“何时委派”决策逻辑，而非单纯依赖模型提供更大的上下文窗口。

**[One Token per Multimodal Evidence: Latent Memory for Resource-Constrained QA](https://huggingface.co/papers/2606.10572)**
🔬 突破：推翻了“多模态记忆必须存储原始文本和图像”的假设，提出**潜在空间记忆（Latent Memory）**——将每个文本或图像证据压缩为一个高维向量（即一个token）。在资源受限设备（如手机）上，相比传统RAG方案（如将图像base64编码后传入LLM），Latent Memory将token消耗降低了99.7%（从平均1500 tokens/image降至1 token/image），同时QA准确率仅下降2.1%。
⚙️ 工程影响：这对边缘AI部署是颠覆性的。它意味着在手机上运行的Agent可以“记住”大量多模态证据而不耗尽上下文窗口。工程团队需要关注的是：压缩过程本身有计算开销（约50ms/image），且压缩后的向量无法直接用于人类可读的审查——这是一个“可审计性”与“效率”的权衡。

## 🧠 AI/ML Frontier Papers

**[EEVEE: Towards Test-time Prompt Learning in the Real World for Self-Improving Agents](https://huggingface.co/papers/2606.11182)**
🔬 Breakthrough: Overturns the assumption that "test-time prompt learning can only be effective on a single dataset" by introducing a **task router** that dynamically partitions multi-dataset, multi-domain input streams and learns independent prompt configurations for each partition. In tests mixing 10 different domain datasets, EEVEE improved accuracy by 12.3% over single-prompt baselines (e.g., CoT), with the router's additional overhead being only ~5ms/query.
⚙️ Engineering Impact: This means agents deployed in the real world (e.g., customer service bots) no longer need to maintain a separate model instance for each business domain; instead, they achieve "self-adaptation" through a unified routing + multi-prompt pool architecture. Engineering teams need to focus on: the router's classification accuracy (94.7% in the paper) directly determines overall performance, and the prompt pool needs periodic updates to handle data drift.

**[SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long-Horizon Deep Research](https://huggingface.co/papers/2606.09730)**
🔬 Breakthrough: Improves the paradigm of "a single agent handling long-context tasks" by proposing **Delegation Intelligence** — the main agent delegates subtasks to sub-agents, which only return summaries, thereby reducing the main agent's context consumption by approximately 80%. In a deep research task of "writing a report on the application of quantum computing in finance," SearchSwarm's final report quality (assessed by human experts) was 34% higher than a single-agent baseline (e.g., AutoGPT), and completion time was reduced by 42%.
⚙️ Engineering Impact: This directly challenges the engineering direction of "infinite context windows." SearchSwarm demonstrates that through delegation and summarization between agents, limited context windows can handle theoretically unbounded information. Engineering teams need to implement: a "summary fidelity" monitoring mechanism for sub-agents and a "when to delegate" decision logic for the main agent, rather than simply relying on models to provide larger context windows.

**[One Token per Multimodal Evidence: Latent Memory for Resource-Constrained QA](https://huggingface.co/papers/2606.10572)**
🔬 Breakthrough: Overturns the assumption that "multimodal memory must store raw text and images" by proposing **Latent Memory** — compressing each piece of text or image evidence into a high-dimensional vector (i.e., one token). On resource-constrained devices (e.g., phones), compared to traditional RAG schemes (e.g., base64-encoding images and passing them to the LLM), Latent Memory reduced token consumption by 99.7% (from an average of 1500 tokens/image to 1 token/image), while QA accuracy only dropped by 2.1%.
⚙️ Engineering Impact: This is disruptive for edge AI deployment. It means agents running on phones can "remember" large amounts of multimodal evidence without exhausting the context window. Engineering teams need to focus on: the compression process itself has computational overhead (~50ms/image), and the compressed vectors cannot be directly used for human-readable review — a trade-off between "auditability" and "efficiency."

## 🧠 AI/ML 前沿論文

**[EEVEE: Towards Test-time Prompt Learning in the Real World for Self-Improving Agents](https://huggingface.co/papers/2606.11182)**
🔬 突破：推翻了「test-time prompt learning只能在單一資料集上有效」的假設，透過引入一個**任務路由器**，將多資料集、多領域的輸入流動態分區，為每個分區學習獨立的prompt配置。在混合了10個不同領域資料集的測試中，EEVEE比單prompt基線（如CoT）的準確率提升了12.3%，且路由器的額外開銷僅約5ms/query。
⚙️ 工程影響：這意味著部署在真實世界的Agent（如客服機器人）不再需要為每個業務領域單獨維護一個模型實例，而是透過一個統一的路由+多prompt池架構實現「自我適應」。工程團隊需要關注的是：路由器的分類準確率（論文中為94.7%）直接決定了整體性能，且prompt池需要定期更新以應對資料漂移。

**[SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long-Horizon Deep Research](https://huggingface.co/papers/2606.09730)**
🔬 突破：改進了「單一Agent處理長上下文任務」的範式，提出**委託智能（Delegation Intelligence）**——主Agent將子任務委派給子Agent，子Agent只返回摘要，從而將主Agent的上下文消耗降低約80%。在「撰寫一份關於量子計算在金融領域的應用」的深度研究任務中，SearchSwarm的最終報告品質（由人類專家評估）比單Agent基線（如AutoGPT）高出34%，且完成時間縮短了42%。
⚙️ 工程影響：這直接挑戰了「無限上下文窗口」的工程方向。SearchSwarm表明，透過Agent間的委派和摘要，有限的上下文窗口可以處理理論上無邊界的資訊。工程團隊需要實現的是：子Agent的「摘要保真度」監控機制，以及主Agent的「何時委派」決策邏輯，而非單純依賴模型提供更大的上下文窗口。

**[One Token per Multimodal Evidence: Latent Memory for Resource-Constrained QA](https://huggingface.co/papers/2606.10572)**
🔬 突破：推翻了「多模態記憶必須儲存原始文字和圖像」的假設，提出**潛在空間記憶（Latent Memory）**——將每個文字或圖像證據壓縮為一個高維向量（即一個token）。在資源受限設備（如手機）上，相比傳統RAG方案（如將圖像base64編碼後傳入LLM），Latent Memory將token消耗降低了99.7%（從平均1500 tokens/image降至1 token/image），同時QA準確率僅下降2.1%。
⚙️ 工程影響：這對邊緣AI部署是顛覆性的。它意味著在手機上運行的Agent可以「記住」大量多模態證據而不耗盡上下文窗口。工程團隊需要關注的是：壓縮過程本身有計算開銷（約50ms/image），且壓縮後的向量無法直接用於人類可讀的審查——這是一個「可審計性」與「效率」的權衡。
