## 🧠 AI/ML 前沿论文

**[The Cold-Start Safety Gap in LLM Agents](https://huggingface.co/papers/2606.07867)**
🔬 突破：推翻了“LLM Agent在整个对话中安全水平一致”的隐含假设。论文发现Agent在会话开始时的安全漏洞率比执行5个常规任务后高9-52%（7个模型、4个家族的评估结果）。这个“冷启动安全缺口”意味着现有安全评估（通常从第一轮对话开始测试）系统性地高估了Agent的安全性。
⚙️ 工程影响：Agent部署时必须在会话启动阶段实施额外的安全护栏（如强制执行“预热”任务、限制初始工具权限），而非均匀分布安全策略。对生产环境Agent的编排逻辑有直接影响：需要在第一个用户请求前插入至少3-5个“安全校准”任务。

**[Getting Better at Working With You: Compiling User Corrections into Runtime Enforcement for Coding Agents](https://huggingface.co/papers/2606.13174)**
🔬 突破：量化了现有记忆方案（Mem0）的失败率——在真实用户摩擦案例中，Mem0仍有57.5%的用户偏好检查被违反。论文提出的TRACE方案通过将用户修正编译为原子规则并在运行时强制执行，将违规率降至约10%。关键创新在于“编译”而非“记忆”：规则是确定的、可审计的，而非概率性的。
⚙️ 工程影响：对Coding Agent的架构设计有直接指导意义——不应依赖Agent的“学习能力”来记住用户偏好，而应设计一个独立的规则引擎层。这意味着Agent框架需要增加“规则编译”和“运行时强制执行”两个组件，而非仅仅增强记忆模块。

**[ToolSense: A Diagnostic Framework for Auditing Parametric Tool Knowledge in LLMs](https://huggingface.co/papers/2606.12451)**
🔬 突破：揭示了现有工具检索基准（ToolBench）的评估盲区——它们使用“冗长、完全指定的查询”，掩盖了LLM在模糊查询下工具检索能力的不足。论文提出的ToolSense框架通过构造“模糊查询”和“干扰工具”场景，发现当前最佳参数化工具检索方法（将工具编码为虚拟token）在模糊查询下的召回率下降约30%。
⚙️ 工程影响：对Agent系统中工具检索模块的评估标准提出了新要求——不能仅在理想查询下测试，必须包含模糊查询和干扰项。这意味着工具检索的benchmark需要重新设计，且生产环境中需要增加查询消歧机制。

## 🧠 AI/ML Frontier Papers

**[The Cold-Start Safety Gap in LLM Agents](https://huggingface.co/papers/2606.07867)**
🔬 Breakthrough: Overturns the implicit assumption that "LLM Agents maintain consistent safety levels throughout a conversation." The paper finds that the safety vulnerability rate at the start of an Agent conversation is 9-52% higher than after performing 5 routine tasks (evaluated across 7 models from 4 families). This "cold-start safety gap" implies that existing safety evaluations (typically starting from the first conversation turn) systematically overestimate Agent safety.
⚙️ Engineering Impact: Agent deployments must implement additional safety guardrails during the session startup phase (e.g., enforcing "warm-up" tasks, limiting initial tool permissions) rather than distributing safety policies uniformly. This has a direct impact on the orchestration logic of production Agents: at least 3-5 "safety calibration" tasks need to be inserted before the first user request.

**[Getting Better at Working With You: Compiling User Corrections into Runtime Enforcement for Coding Agents](https://huggingface.co/papers/2606.13174)**
🔬 Breakthrough: Quantifies the failure rate of existing memory solutions (Mem0)—in real user friction cases, Mem0 still violates 57.5% of user preference checks. The proposed TRACE scheme reduces the violation rate to approximately 10% by compiling user corrections into atomic rules and enforcing them at runtime. The key innovation is "compilation" rather than "memory": rules are deterministic and auditable, not probabilistic.
⚙️ Engineering Impact: Provides direct guidance for the architecture design of Coding Agents—do not rely on the Agent's "learning ability" to remember user preferences; instead, design an independent rule engine layer. This means Agent frameworks need to add two components: "rule compilation" and "runtime enforcement," rather than merely enhancing the memory module.

**[ToolSense: A Diagnostic Framework for Auditing Parametric Tool Knowledge in LLMs](https://huggingface.co/papers/2606.12451)**
🔬 Breakthrough: Reveals a blind spot in existing tool retrieval benchmarks (ToolBench)—they use "verbose, fully specified queries," masking the LLM's deficiency in tool retrieval under ambiguous queries. The proposed ToolSense framework, by constructing "ambiguous query" and "distractor tool" scenarios, finds that the recall of current best parametric tool retrieval methods (encoding tools as virtual tokens) drops by approximately 30% under ambiguous queries.
⚙️ Engineering Impact: Imposes new requirements on the evaluation criteria for tool retrieval modules in Agent systems—testing must not only occur under ideal queries but must also include ambiguous queries and distractors. This means tool retrieval benchmarks need to be redesigned, and production environments require query disambiguation mechanisms.
