## 🧠 AI/ML 前沿论文

**[RedAct: Redacting Agent Capability Traces for Procedural Skill Protection](https://huggingface.co/papers/2606.10813)**
🔬 突破：推翻了“Agent执行轨迹是安全日志、无需保护”的假设。论文构建了CapTraceBench（75个长时任务、154个技能、7个领域），证明从执行轨迹中可恢复出90%+的关键参数（如API密钥、业务规则阈值、错误恢复策略），且无需访问模型权重。
⚙️ 工程影响：这意味着部署Agent时，不能简单地将执行轨迹视为“调试日志”公开。RedAct提出的“能力痕迹编辑”技术（在保留轨迹可审计性的同时，删除可被逆向的技能参数）将成为Agent运维平台的标配功能，类似日志脱敏工具（如Vector的`redact` transform）但针对Agent特有的工具调用链。

**[MBench: A Comprehensive Benchmark on Memory Capability for Video World Models](https://huggingface.co/papers/2606.00793)**
🔬 突破：揭示了当前视频世界模型（如Sora、CogVideo）的一个根本缺陷——它们擅长生成视觉上逼真的帧，但在维持超过10秒的长期一致性上失败率超过60%。MBench测试了模型在“物体消失后重新出现是否保持相同外观”、“场景切换后物体位置是否连续”等记忆能力，发现所有现有模型在超过20秒的时间跨度上，记忆一致性指标（MCI）低于0.4（满分1.0）。
⚙️ 工程影响：这意味着“用视频生成模型做世界模型”的路线需要重新评估。对于需要长期记忆的应用（如自动驾驶仿真、游戏NPC行为模拟），不能直接使用现有视频生成模型，而需要引入显式的记忆模块（如NeRF的3D场景表示）来维持一致性。

**[LLM Agents Can See Code Repositories](https://huggingface.co/papers/2606.14061)**
🔬 突破：推翻了“Agent只能通过文本理解代码库”的假设。论文发现，当给Agent提供代码库的“可视化表示”（如文件夹层级树、依赖关系图、代码热力图）时，Agent在代码理解任务上的准确率提升27%，在bug定位任务上的速度提升3倍。关键洞察是：人类开发者通过视觉结构（如IDE的项目导航）快速定位代码，而现有Agent只能逐文件阅读。
⚙️ 工程影响：这意味着下一代代码Agent（如GitHub Copilot、Cursor）需要集成“代码库可视化”作为输入模态，而非仅依赖文本。具体实现上，可以将代码库的AST结构渲染为图像，作为多模态LLM的输入，让Agent“看到”代码的组织结构而非仅“读到”代码文本。

## 🧠 AI/ML Frontier Papers

**[RedAct: Redacting Agent Capability Traces for Procedural Skill Protection](https://huggingface.co/papers/2606.10813)**
🔬 Breakthrough: Overturns the assumption that "Agent execution traces are safety logs and do not need protection." The paper constructs CapTraceBench (75 long-term tasks, 154 skills, 7 domains), proving that 90%+ of key parameters (e.g., API keys, business rule thresholds, error recovery strategies) can be recovered from execution traces without accessing model weights.
⚙️ Engineering Impact: This means that when deploying Agents, execution traces cannot simply be made public as "debug logs." The "capability trace redaction" technique proposed by RedAct (removing reversible skill parameters while preserving trace auditability) will become a standard feature of Agent operation platforms, similar to log sanitization tools (e.g., Vector's `redact` transform) but tailored to Agent-specific tool call chains.

**[MBench: A Comprehensive Benchmark on Memory Capability for Video World Models](https://huggingface.co/papers/2606.00793)**
🔬 Breakthrough: Reveals a fundamental flaw in current video world models (e.g., Sora, CogVideo)—they excel at generating visually realistic frames but fail to maintain long-term consistency beyond 10 seconds with a failure rate exceeding 60%. MBench tests models on memory capabilities such as "whether an object reappears with the same appearance after disappearing" and "whether object positions remain continuous after scene transitions," finding that all existing models have a Memory Consistency Index (MCI) below 0.4 (out of 1.0) for time spans exceeding 20 seconds.
⚙️ Engineering Impact: This means the approach of "using video generation models as world models" needs reassessment. For applications requiring long-term memory (e.g., autonomous driving simulation, game NPC behavior simulation), existing video generation models cannot be used directly; explicit memory modules (e.g., NeRF's 3D scene representation) must be introduced to maintain consistency.

**[LLM Agents Can See Code Repositories](https://huggingface.co/papers/2606.14061)**
🔬 Breakthrough: Overturns the assumption that "Agents can only understand codebases through text." The paper finds that when Agents are provided with "visual representations" of codebases (e.g., folder hierarchy trees, dependency graphs, code heatmaps), their accuracy on code understanding tasks improves by 27%, and bug localization speed increases by 3x. The key insight is that human developers quickly locate code through visual structures (e.g., IDE project navigation), while existing Agents can only read files one by one.
⚙️ Engineering Impact: This means the next generation of code Agents (e.g., GitHub Copilot, Cursor) needs to integrate "codebase visualization" as an input modality, rather than relying solely on text. Specifically, the AST structure of a codebase can be rendered as an image and used as input to a multimodal LLM, allowing the Agent to "see" the organizational structure of the code rather than just "read" the code text.

⚙️ 工程影響：這意味著「用影片生成模型做世界模型」的路線需要重新評估。對於需要長期記憶的應用（如自動駕駛模擬、遊戲NPC行為模擬），不能直接使用現有影片生成模型，而需要引入顯式的記憶模組（如NeRF的3D場景表示）來維持一致性。
