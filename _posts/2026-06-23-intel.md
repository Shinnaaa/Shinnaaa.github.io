## 🧠 AI/ML 前沿论文

**[KaLM-Reranker-V1: Fast but Not Late Interaction for Compressed Document Reranking](https://huggingface.co/papers/2606.22807)**
🔬 突破：推翻了“高质量重排序必须联合编码查询和文档”的假设。现有重排序器（如Cohere Rerank、BGE-Reranker）需要将查询和文档拼接后一起编码，导致计算量随文档数线性增长。KaLM-Reranker-V1通过编码器-解码器架构，将文档预编码为“Matryoshka embedding”（一种分层压缩的向量表示），查询单独编码，然后在解码器中通过“非延迟交互”计算相关性。在BEIR基准上，KaLM-Reranker-V1在保持与Cohere Rerank v3相当准确率（-0.3%）的同时，将重排序延迟从每文档15ms降至2ms（7.5倍加速），且支持文档预缓存。
⚙️ 工程影响：这意味着RAG流水线可以“一次编码，多次重排序”——文档只需在入库时编码一次，后续不同查询的重排序只需计算查询编码和轻量级交互，将重排序从“查询时计算”变为“查询时检索”，大幅降低在线推理成本。

**[Manifold Bandits: Bayesian Curriculum Learning over the Latent Geometry of Large Language Models](https://huggingface.co/papers/2606.19750)**
🔬 突破：改进了RLHF中“课程学习”的假设——现有方法（如PAL、Curriculum RL）将问题选择视为独立臂的赌博机问题，忽略了问题之间的结构相似性。Manifold Bandits将LLM的潜在空间建模为黎曼流形，通过“流形上的贝叶斯优化”动态选择下一个训练问题。在MATH和GSM8K上，相比随机采样和现有课程学习方法，Manifold Bandits将收敛速度提升2.1倍（达到相同准确率所需训练步数减少52%），且最终准确率提升3.4%。
⚙️ 工程影响：这意味着RL训练的效率瓶颈可能不在算法（PPO/GRPO），而在数据采样策略。Manifold Bandits提供了一个即插即用的采样模块，可以集成到现有RL训练框架（如TRL、slime）中，无需修改训练循环。

**[EvoEmbedding: Evolvable Representations for Long-Context Retrieval and Agentic Memory](https://huggingface.co/papers/2606.21649)**
🔬 突破：推翻了“嵌入是静态的”假设——现有嵌入模型（如OpenAI Embeddings、E5）对同一段文本始终生成相同的向量，忽略了上下文和时序。EvoEmbedding通过维护一个“持续更新的潜在记忆”，在顺序处理输入时动态调整嵌入。在LongBench和NarrativeQA上，EvoEmbedding在长上下文检索任务中相比静态嵌入（E5-mistral-7b）将Recall@5提升12.7%，在Agent记忆检索任务中提升18.3%。
⚙️ 工程影响：这意味着Agent的“记忆”可以从“向量数据库+固定嵌入”升级为“动态嵌入+状态跟踪”。EvoEmbedding的“可演化嵌入”可以直接替代现有RAG系统中的嵌入模型，无需修改检索流水线，即可让Agent感知到“上下文变化”（如对话历史、文档更新）。

## 🧠 AI/ML Frontier Papers

**[KaLM-Reranker-V1: Fast but Not Late Interaction for Compressed Document Reranking](https://huggingface.co/papers/2606.22807)**
🔬 Breakthrough: Overturns the assumption that "high-quality reranking requires joint encoding of query and document." Existing rerankers (like Cohere Rerank, BGE-Reranker) need to concatenate the query and document and encode them together, causing computation to grow linearly with the number of documents. KaLM-Reranker-V1 uses an encoder-decoder architecture to pre-encode documents into "Matryoshka embeddings" (a hierarchically compressed vector representation), encodes the query separately, and then computes relevance in the decoder through "non-late interaction." On the BEIR benchmark, KaLM-Reranker-V1 maintains accuracy comparable to Cohere Rerank v3 (-0.3%) while reducing reranking latency from 15ms per document to 2ms (7.5x speedup), and supports document pre-caching.
⚙️ Engineering Impact: This means RAG pipelines can "encode once, rerank multiple times" — documents only need to be encoded once when ingested, and subsequent reranking for different queries only requires computing the query encoding and lightweight interaction, shifting reranking from "compute at query time" to "retrieve at query time," significantly reducing online inference costs.

**[Manifold Bandits: Bayesian Curriculum Learning over the Latent Geometry of Large Language Models](https://huggingface.co/papers/2606.19750)**
🔬 Breakthrough: Improves the "curriculum learning" assumption in RLHF — existing methods (like PAL, Curriculum RL) treat problem selection as a bandit problem with independent arms, ignoring the structural similarity between problems. Manifold Bandits models the LLM's latent space as a Riemannian manifold and dynamically selects the next training problem through "Bayesian optimization on the manifold." On MATH and GSM8K, compared to random sampling and existing curriculum learning methods, Manifold Bandits improves convergence speed by 2.1x (52% fewer training steps to reach the same accuracy) and increases final accuracy by 3.4%.
⚙️ Engineering Impact: This means the efficiency bottleneck in RL training might not be the algorithm (PPO/GRPO), but the data sampling strategy. Manifold Bandits provides a plug-and-play sampling module that can be integrated into existing RL training frameworks (like TRL, slime) without modifying the training loop.

**[EvoEmbedding: Evolvable Representations for Long-Context Retrieval and Agentic Memory](https://huggingface.co/papers/2606.21649)**
🔬 Breakthrough: Overturns the assumption that "embeddings are static" — existing embedding models (like OpenAI Embeddings, E5) always generate the same vector for the same text, ignoring context and temporality. EvoEmbedding dynamically adjusts embeddings by maintaining a "continuously updated latent memory" while processing input sequentially. On LongBench and NarrativeQA, EvoEmbedding improves Recall@5 by 12.7% in long-context retrieval tasks and by 18.3% in Agent memory retrieval tasks compared to static embeddings (E5-mistral-7b).
⚙️ Engineering Impact: This means Agent "memory" can be upgraded from "vector database + fixed embeddings" to "dynamic embeddings + state tracking." EvoEmbedding's "evolvable embeddings" can directly replace the embedding model in existing RAG systems without modifying the retrieval pipeline, allowing the Agent to perceive "context changes" (like conversation history, document updates).

## 🧠 AI/ML 前沿論文

**[KaLM-Reranker-V1: Fast but Not Late Interaction for Compressed Document Reranking](https://huggingface.co/papers/2606.22807)**
🔬 突破：推翻了「高品質重排序必須聯合編碼查詢和文件」的假設。現有重排序器（如Cohere Rerank、BGE-Reranker）需要將查詢和文件拼接後一起編碼，導致計算量隨文件數線性增長。KaLM-Reranker-V1透過編碼器-解碼器架構，將文件預編碼為「Matryoshka embedding」（一種分層壓縮的向量表示），查詢單獨編碼，然後在解碼器中透過「非延遲互動」計算相關性。在BEIR基準上，KaLM-Reranker-V1在保持與Cohere Rerank v3相當準確率（-0.3%）的同時，將重排序延遲從每文件15ms降至2ms（7.5倍加速），且支援文件預快取。
⚙️ 工程影響：這意味著RAG流水線可以「一次編碼，多次重排序」——文件只需在入庫時編碼一次，後續不同查詢的重排序只需計算查詢編碼和輕量級互動，將重排序從「查詢時計算」變為「查詢時檢索」，大幅降低線上推理成本。

**[Manifold Bandits: Bayesian Curriculum Learning over the Latent Geometry of Large Language Models](https://huggingface.co/papers/2606.19750)**
🔬 突破：改進了RLHF中「課程學習」的假設——現有方法（如PAL、Curriculum RL）將問題選擇視為獨立臂的賭博機問題，忽略了問題之間的結構相似性。Manifold Bandits將LLM的潛在空間建模為黎曼流形，透過「流形上的貝葉斯最佳化」動態選擇下一個訓練問題。在MATH和GSM8K上，相比隨機取樣和現有課程學習方法，Manifold Bandits將收斂速度提升2.1倍（達到相同準確率所需訓練步數減少52%），且最終準確率提升3.4%。
⚙️ 工程影響：這意味著RL訓練的效率瓶頸可能不在演算法（PPO/GRPO），而在資料取樣策略。Manifold Bandits提供了一個即插即用的取樣模組，可以整合到現有RL訓練框架（如TRL、slime）中，無需修改訓練循環。

**[EvoEmbedding: Evolvable Representations for Long-Context Retrieval and Agentic Memory](https://huggingface.co/papers/2606.21649)**
🔬 突破：推翻了「嵌入是靜態的」假設——現有嵌入模型（如OpenAI Embeddings、E5）對同一段文字始終產生相同的向量，忽略了上下文和時序。EvoEmbedding透過維護一個「持續更新的潛在記憶」，在順序處理輸入時動態調整嵌入。在LongBench和NarrativeQA上，EvoEmbedding在長上下文檢索任務中相比靜態嵌入（E5-mistral-7b）將Recall@5提升12.7%，在Agent記憶檢索任務中提升18.3%。
⚙️ 工程影響：這意味著Agent的「記憶」可以從「向量資料庫+固定嵌入」升級為「動態嵌入+狀態追蹤」。EvoEmbedding的「可演化嵌入」可以直接替代現有RAG系統中的嵌入模型，無需修改檢索流水線，即可讓Agent感知到「上下文變化」（如對話歷史、文件更新）。
