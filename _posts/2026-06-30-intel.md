## 🧠 AI/ML 前沿论文

**[Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent](https://huggingface.co/papers/2606.30616)**
🔬 突破：推翻“更大模型=更好性能”的假设——一个35B MoE Agent模型（Agents-A1）通过将Agent轨迹扩展到平均45K token，在多个基准上达到了万亿参数模型的性能。核心发现是：模型能力不仅取决于参数数量，还取决于“Agent视野”（agent horizon）——即Agent在一次任务中能处理的多步推理和工具调用序列的长度。对比GPT-4（1.8T参数）在相同任务上的表现，Agents-A1在工具使用准确率上仅低3%，但推理成本降低50倍。
⚙️ 工程影响：这意味着部署Agent时，**优化推理轨迹长度比增加模型参数更有效**。对于团队正在构建的Agent系统，应该优先设计长上下文（>32K token）的推理管道，而非等待更大参数的模型发布。具体来说，需要重新设计Agent的“记忆管理”——如何让Agent在45K token的轨迹中保持推理一致性，而不是简单地增加上下文窗口。

**[TACO: Tool-Augmented Credit Optimization for Agentic Tool Use](https://huggingface.co/papers/2606.30251)**
🔬 突破：解决了Agent工具使用的“信用分配”问题——现有强化学习方法（如GRPO）只能给整个轨迹一个奖励，无法区分“有用的工具调用”和“冗余/误导性的工具调用”。TACO引入了两个耦合的优势通道：一个评估工具调用对最终答案的贡献（工具级信用），另一个评估推理步骤的正确性（推理级信用）。在视觉问答任务上，TACO将工具使用准确率从78.3%提升到91.2%，同时减少了32%的不必要工具调用。
⚙️ 工程影响：这意味着训练Agent时，**不能只奖励最终答案正确，还要奖励“正确的工具使用策略”**。对于团队正在微调的Agent模型，应该将TACO的“双通道信用分配”集成到训练流程中，而不是使用标准的GRPO。这需要修改奖励模型，使其能评估每个工具调用的边际贡献。

**[How Good Can Linear Models Be for Time-Series Forecasting?](https://huggingface.co/papers/2606.27282)**
🔬 突破：推翻“时间序列预测需要复杂架构”的假设——通过精细调优预处理（上下文长度、局部归一化、正则化、数据增强），简单的Ridge回归在Monash基准上达到了与PatchTST（Transformer）相当的预测性能（平均sMAPE仅高1.2%），但训练速度快1000倍。核心发现是：**预处理策略比模型架构更重要**——错误的上下文长度选择会导致Transformer比Ridge更差。
⚙️ 工程影响：这意味着团队在构建时间序列预测系统时，**应该先花80%的精力调优预处理，而不是直接上Transformer**。具体来说，需要建立一个“预处理搜索空间”（上下文长度、归一化窗口、正则化强度），用Ridge回归作为baseline，只有当Ridge无法达到目标精度时，才考虑使用复杂模型。这可以节省大量的GPU训练成本。

## 🧠 AI/ML Frontier Papers

**[Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent](https://huggingface.co/papers/2606.30616)**
🔬 Breakthrough: Overturns the assumption that "bigger model = better performance." A 35B MoE Agent model (Agents-A1), by scaling its Agent trajectory to an average of 45K tokens, achieves performance comparable to trillion-parameter models on multiple benchmarks. The core finding is that model capability depends not only on parameter count but also on "agent horizon"—the length of multi-step reasoning and tool-calling sequences an Agent can handle in a single task. Compared to GPT-4 (1.8T parameters) on the same tasks, Agents-A1 is only 3% lower in tool-use accuracy but reduces inference cost by 50x.
⚙️ Engineering Impact: This means that when deploying Agents, **optimizing reasoning trajectory length is more effective than increasing model parameters**. For teams building Agent systems, priority should be given to designing long-context (>32K token) reasoning pipelines rather than waiting for larger model releases. Specifically, Agent "memory management" needs redesign—how to maintain reasoning consistency across a 45K token trajectory, rather than simply increasing the context window.

**[TACO: Tool-Augmented Credit Optimization for Agentic Tool Use](https://huggingface.co/papers/2606.30251)**
🔬 Breakthrough: Solves the "credit assignment" problem in Agent tool use. Existing reinforcement learning methods (e.g., GRPO) can only assign a single reward to the entire trajectory, unable to distinguish between "useful tool calls" and "redundant/misleading tool calls." TACO introduces two coupled advantage channels: one evaluates a tool call's contribution to the final answer (tool-level credit), and the other evaluates the correctness of reasoning steps (reasoning-level credit). On visual question answering tasks, TACO improves tool-use accuracy from 78.3% to 91.2% while reducing unnecessary tool calls by 32%.
⚙️ Engineering Impact: This means that when training Agents, **you cannot reward only the final correct answer; you must also reward "correct tool-use strategies."** For teams fine-tuning Agent models, TACO's "dual-channel credit assignment" should be integrated into the training pipeline instead of using standard GRPO. This requires modifying the reward model to evaluate the marginal contribution of each tool call.

**[How Good Can Linear Models Be for Time-Series Forecasting?](https://huggingface.co/papers/2606.27282)**
🔬 Breakthrough: Overturns the assumption that "time-series forecasting requires complex architectures." Through finely tuned preprocessing (context length, local normalization, regularization, data augmentation), simple Ridge regression achieves predictive performance comparable to PatchTST (Transformer) on the Monash benchmark (average sMAPE only 1.2% higher), but is 1000x faster to train. The core finding is: **preprocessing strategy matters more than model architecture**—a wrong choice of context length can make Transformer perform worse than Ridge.
⚙️ Engineering Impact: This means that when building time-series forecasting systems, **teams should spend 80% of their effort tuning preprocessing rather than jumping straight to Transformers.** Specifically, a "preprocessing search space" (context length, normalization window, regularization strength) should be established, using Ridge regression as a baseline. Only when Ridge fails to meet target accuracy should complex models be considered. This can save significant GPU training costs.

## 🧠 AI/ML 前沿论文

**[Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent](https://huggingface.co/papers/2606.30616)**
🔬 突破：推翻了“更大模型=更好性能”的假设——一个35B MoE Agent模型（Agents-A1）通过将Agent轨迹扩展到平均45K token，在多个基准上达到了万亿参数模型的性能。核心发现是：模型能力不仅取决于参数数量，还取决于“Agent视野”（agent horizon）——即Agent在一次任务中能处理的多步推理和工具调用序列的长度。对比GPT-4（1.8T参数）在相同任务上的表现，Agents-A1在工具使用准确率上仅低3%，但推理成本降低50倍。
⚙️ 工程影响：这意味着部署Agent时，**优化推理轨迹长度比增加模型参数更有效**。对于团队正在构建的Agent系统，应该优先设计长上下文（>32K token）的推理管道，而非等待更大参数的模型发布。具体来说，需要重新设计Agent的“记忆管理”——如何让Agent在45K token的轨迹中保持推理一致性，而不是简单地增加上下文窗口。

**[TACO: Tool-Augmented Credit Optimization for Agentic Tool Use](https://huggingface.co/papers/2606.30251)**
🔬 突破：解决了Agent工具使用的“信用分配”问题——现有强化学习方法（如GRPO）只能给整个轨迹一个奖励，无法区分“有用的工具调用”和“冗余/误导性的工具调用”。TACO引入了两个耦合的优势通道：一个评估工具调用对最终答案的贡献（工具级信用），另一个评估推理步骤的正确性（推理级信用）。在视觉问答任务上，TACO将工具使用准确率从78.3%提升到91.2%，同时减少了32%的不必要工具调用。
⚙️ 工程影响：这意味着训练Agent时，**不能只奖励最终答案正确，还要奖励“正确的工具使用策略”**。对于团队正在微调的Agent模型，应该将TACO的“双通道信用分配”集成到训练流程中，而不是使用标准的GRPO。这需要修改奖励模型，使其能评估每个工具调用的边际贡献。

**[How Good Can Linear Models Be for Time-Series Forecasting?](https://huggingface.co/papers/2606.27282)**
🔬 突破：推翻了“时间序列预测需要复杂架构”的假设——通过精细调优预处理（上下文长度、局部归一化、正则化、数据增强），简单的Ridge回归在Monash基准上达到了与PatchTST（Transformer）相当的预测性能（平均sMAPE仅高1.2%），但训练速度快1000倍。核心发现是：**预处理策略比模型架构更重要**——错误的上下文长度选择会导致Transformer比Ridge更差。
⚙️ 工程影响：这意味着团队在构建时间序列预测系统时，**应该先花80%的精力调优预处理，而不是直接上Transformer**。具体来说，需要建立一个“预处理搜索空间”（上下文长度、归一化窗口、正则化强度），用Ridge回归作为baseline，只有当Ridge无法达到目标精度时，才考虑使用复杂模型。这可以节省大量的GPU训练成本。
